# StudySpot

## 📌 프로젝트 소개
StudySpot은 **로그인 기반의 스터디룸 예약 시스템**으로,  
사용자가 자신의 동네에 있는 스터디룸을 조회하고 날짜별로 **1시간 단위의 시간 슬롯을 선택해 예약**할 수 있는 라이트한 웹 서비스입니다.

이 프로젝트는 **두 명이서 진행하는 팀 프로젝트**이며,  
두 명 모두 **프론트엔드와 백엔드를 구분하지 않고 풀스택으로 전반적인 기능을 함께 구현**하는 것을 목표로 하고 있습니다.

복잡한 기능보다는,  
👉 핵심 기능을 안정적으로 구현하고  
👉 실제 협업과 개발 흐름에 익숙해지는 데 초점을 맞춘 연습용 프로젝트입니다.

---

## 👥 팀 구성
- 총 인원: 2명  
- 역할 분담:  
  - 이지수(PM): 프론트앤드, 백앤드
  - 문이화 :프론트앤드, 백엔드 

---

## ✨ 주요 기능
- 회원 로그인 기능
- 스터디룸 목록 조회
- 날짜별 스터디룸 조회
- 1시간 단위 예약 슬롯 선택
- 예약 상태 관리  
  - `예약됨 → 이용 완료`
- 예약 시간이 지나면 스케줄러를 통해 상태 자동 변경
- 위치 정보는 참고용으로 간단히 제공

※ 연속 예약, 복잡한 시간 계산 등은 제외하여  
소규모 팀에서도 안정적으로 구현할 수 있는 범위로 설계했습니다.

---

## 🛠️ 기술 스택 (예정)
- Frontend: HTML / CSS / JavaScript
- Backend: Java (또는 Spring 기반)
- Database: MySQL
- Version Control: Git / GitHub
- Collaboration: GitHub + SourceTree

(※ 학습 및 진행 상황에 따라 변경될 수 있습니다.)

---

## 🎯 프로젝트 목적
- Java 및 웹 개발 전반에 대한 이해를 바탕으로, **로그인·예약·상태 관리 등 실제 서비스에서 자주 사용되는 기능 흐름을 구현**
- GitHub를 활용한 **2인 협업 프로젝트를 통해 브랜치 전략, 커밋 규칙, 코드 리뷰 등 협업 과정 경험**
- 단순 기능 나열이 아닌, **기능 범위를 명확히 설정하고 안정적으로 완성하는 개발 경험**
- 프론트엔드와 백엔드를 모두 다루는 **풀스택 개발 흐름을 직접 경험하며 시스템 구조에 대한 이해도 향상**
- 소규모 팀 환경에서 요구되는 **문제 분해 능력과 구현 우선순위 설정 능력 강화**
